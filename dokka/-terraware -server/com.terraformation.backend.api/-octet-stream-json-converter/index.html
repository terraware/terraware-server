OctetStreamJsonConverter

@Named
class OctetStreamJsonConverter(objectMapper: ObjectMapper) : HttpMessageConverter<Any> (source)

Workaround for a bug in swagger-ui that causes it to leave off the content type of JSON parts of multipart/form-data POST requests. This causes the parts to be treated as application/octet-stream (the default content type) and Spring has no way of knowing that it's really JSON.

To prevent this converter from unintentionally converting objects that should really be treated as binary data, it only works if the target class is annotated with JsonDeserialize.
