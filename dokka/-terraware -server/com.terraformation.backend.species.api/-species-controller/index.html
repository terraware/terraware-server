class SpeciesController(speciesService: SpeciesService, speciesStore: SpeciesStore)(source)

Species management endpoints.

Implementation note: The names of the methods and payload classes are unusual here because there are some existing species management endpoints in ValuesController, and those endpoints are already using the obvious names. The OpenAPI schema doesn't allow duplicate names, so if we reused names such as CreateSpeciesRequestPayload here, the schema generator would pick one and ignore the other. We can't change the names in ValuesController because the seed bank app uses the existing names to look up the payload definitions.

Once we've updated the seed bank client code to use these endpoints, we can get rid of the equivalents in ValuesController and update the names here to conform to our usual naming conventions.
