requirePermissions

fun requirePermissions(user: TerrawareUser = currentUser(), func: PermissionRequirements.() -> Unit)(source)

Requires a user to have a certain set of permissions.

This allows us to use a declarative code style to do permission checks, like:

requirePermissions { createDevice(facilityId) }
Content copied to clipboard

If the user doesn't have the requested permissions, this method will throw an appropriate exception.

The permission checks are just the methods of PermissionRequirements; see that class for the complete list of available ones or to add new ones.

Parameters

user

Which user's permission to check. In the vast majority of cases, this should be the current user, which is the default value; you will generally want to omit this parameter.

func

Permission checks. This is called with a PermissionRequirements receiver, so inside the function, this refers to the PermissionRequirements rather than the calling class. See the Kotlin docs for more information about how that works.
